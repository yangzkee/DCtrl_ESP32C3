ESP32-C3 循线项目硬件框架搭建清单

目标不是手把手教每一个 IO 口怎么配，而是把主控、传感器、底盘、供电、烧录和最小通信链路搭到可写算法的程度。全部打勾后，读者就有一个能承载核心代码、主循环和控制逻辑的工程底座。

  • ✓

    只定义基础框架和合格线，不讲循线算法。

  • ✓

    只要求关键链路可验证，不绑定某一块开发板的具体引脚。

  • ✓

    先把硬件底座搭稳，再让读者自由发挥上层代码。

三类硬件角色

ESP32-C3 主控 负责运行工程、读取数据、输出控制指令。
⇅ 串口通信
8 路红外巡线模块 只提供黑线/白底的传感数据。
⇄ 运动指令
串口底盘模块 只接收安全低速、零速等底盘命令。

按这 6 步搭出可写算法的项目底座

每一步都要能被观察或复现。看起来接好了不算，能稳定烧录、能收发、能停止、数据会变化，才算通过。

✓

1. 主控与模块角色确认

  • ESP32-C3 只做主控，承载工程和你要写的核心代码。
  • 8 路红外模块只提供巡线传感数据。
  • 串口底盘模块只接收运动指令，不承担算法判断。
完成标准

三者职责清楚，没有把算法、传感器、底盘协议混成一个不可拆分的黑盒。

✓

2. 供电与共地确认

  • 电机和底盘供电稳定，ESP32-C3 不被电机电流拖垮。
  • ESP32-C3、底盘模块、红外模块必须共地。
  • USB 调试供电和电机供电之间避免反灌和干扰。
完成标准

电机动作时 ESP32-C3 不重启，USB 日志不中断，模块读数不随机跳变。

✓

3. 串口链路确认

  • 底盘 UART：ESP32 TX 接底盘 RX，ESP32 RX 接底盘 TX。
  • 传感器 UART：ESP32 TX 接模块 RX，ESP32 RX 接模块 TX。
  • 如果外设是 5V TTL，ESP32-C3 侧必须做 3.3V 电平保护。
完成标准

底盘串口和传感器串口能独立收发，互不占用，排查时能单独断开其中一路。

✓

4. 固件工程能构建和烧录

  • ESP-IDF 环境可用，能选择 ESP32-C3 目标。
  • 工程能稳定完成 build，不依赖临时手工补文件。
  • USB 烧录后设备能启动，并能看到启动日志。
完成标准

读者有一个可反复编译、烧录、看日志的工程壳，后续改主循环不会先卡在环境问题上。

✓

5. 底盘最小动作验证

  • 写算法前，先验证底盘能接收一条安全低速指令。
  • 低速动作要短、可控，测试区域要留出安全空间。
  • 零速或停止指令必须比前进指令更优先验证。
完成标准

能让底盘低速动一下，也能可靠停下；重复测试时行为一致。

✓

6. 传感器最小读数验证

  • 写算法前，先确认 8 路红外数据会随黑线/白底变化。
  • 用遮挡、放线、移开线的方式观察每一路是否有响应。
  • 记录清楚当前模块里黑线和白底分别对应什么电平或数值。
完成标准

至少能看到 8 路状态随环境变化，并能分清黑线和白底语义。

当以下全部打勾，就可以开始写算法

  • 工程能 build、能 USB 烧录、能看启动日志。
  • ESP32-C3、底盘、传感器供电稳定且共地。
  • 底盘串口能让车低速动作，也能可靠停止。
  • 传感器串口能读到 8 路数据变化。
  • 两条串口链路互不冲突，能分别排查。

这页不要求你完成什么

  • 不要求固定使用某一组 GPIO。
  • 不要求写出循线算法。
  • 不要求完成参数调试页面。
  • 不要求复刻现有项目的内部实现。
