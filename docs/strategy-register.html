策略清单

S01目标地图边界

运动范围
策略当前只处理一圈手工贴出的黑色胶带循线，不处理迷宫、岔路、路线记忆和路口选路。
触发条件进入 AUTO_RUNNING 后，传感器持续读取黑线并执行闭环纠偏。
实现位置components/control/line_trace_policy.c
验证方法一键自检后开始循线，小车应沿单圈黑线运行；遇到岔路类场景不承诺选路。

S02模块边界

运动架构
策略line_trace_policy 只做策略判断；UART、BLE、NVS、板级引脚留在外层模块。
触发条件任何新的循线、找线、速度或转向策略，都先判断是否属于 policy 层。
实现位置components/control/line_trace_policy.c、components/control/line_trace_controller.c
验证方法S3/C3 双目标构建通过；策略修改不需要改 UART 或 BLE 代码。

S03板级迁移

兼容
策略ESP32-S3 与 ESP32-C3 差异集中在板级 profile，不把芯片差异写进业务策略。
当前 C3 接线底盘 UART1 TX GPIO4 / RX GPIO5 / 115200；循线模块 UART0 TX GPIO2 / RX GPIO3 / 115200；C3 控制台使用 USB Serial/JTAG。
实现位置components/board_profile、sdkconfig.esp32c3
验证方法./scripts/idf.sh -B build-esp32c3 -DSDKCONFIG=sdkconfig.esp32c3 build

S04循线传感器读取

传感器
策略八路循线模块使用 USART 数字模式。发送 $0,0,1#，解析 $D,x1:...,x8:...#。黑线为 0，白底为 1，固件默认把 0 转成有效线 bit。
取舍不用 IO 直读，因为占用 8 个 GPIO；I2C 暂不采用，因为只能读数字值且地址/寄存器细节未稳定。
实现位置components/line_sensor/line_sensor_uart.c
验证方法改变黑线/白线，telemetry 中 line.bits、line.offset、line_uart.last_frame 应变化。

S05底盘速度协议

底盘
策略底盘使用 DFLink Motion_Velocity。linear_mm_s 是直接发给底盘的协议速度值，不保证等同真实速度。angular_mdeg_s 表示单条速度指令中的 Z 角度增量，不是持续角速度。
实测事实Vx=0.5 是已验证的舒适手动/诊断速度；自动循线最高档当前为 Vx=1.0。
实现位置components/chassis/chassis_uart.c、components/chassis/include/chassis_uart.h
验证方法BLE 发送 Cxyz500rx，检查底盘回传和校验。

S06连续发送运动指令

底盘
策略底盘需要持续收到速度指令才会持续运动。控制任务按 control.loop_period_ms 周期持续发送当前运动命令。
默认周期20 ms
实现位置components/control/line_trace_controller.c
验证方法进入 AUTO_RUNNING 后，telemetry 中 motion 应持续更新，底盘连续运动。

S07运动状态机

安全状态
策略车端主运动状态包括 BOOT_INIT、SAFE_IDLE、PARAM_TUNING、MANUAL_TEST、AUTO_ARMED、AUTO_RUNNING、FAULT，OTA 维护状态单独隔离。
触发条件调参、手动测试、自动准备、自动运行、故障和 OTA 必须通过状态转换控制。
实现位置components/vehicle_state/vehicle_state.c
验证方法通过 BLE JSON 发送 arm_auto、start_auto、stop，观察 telemetry 的 vehicle.motion_state。

S08停车调参

安全调试
策略无线调参采用停车调参。只读 telemetry 不影响车运行；写参数、保存参数、恢复默认必须进入 PARAM_TUNING。
小程序例外小程序短帧 S<kp>,<ki>,<kd>,<gear> 是现场简化调参入口，仅用于 PID 和档位。
实现位置components/debug_protocol/debug_protocol.c、components/debug_server/debug_server_ble.c
验证方法AUTO_RUNNING 中 JSON set_param 应返回需要进入调参的错误。

S09BLE 优先调试

无线
策略BLE Low Energy 是默认调试和小程序参数通道。Wi-Fi 默认关闭以降低功耗和供电压力。
协议小程序短帧使用 G、S...、N...；长诊断使用 BLE JSON/分包。
实现位置components/debug_server/debug_server_ble.c、wechat-miniprogram/
验证方法使用微信小程序或 tools/ble_debug_tester.swift 读取 G。

S10Wi-Fi 后备与 OTA

维护安全
策略Wi-Fi SoftAP 只作为后备诊断和无线烧录入口。通过 BLE W1 手动开启。OTA 进入维护状态、强制停车、写入另一个 OTA app 槽并重启。
边界小程序改 PID/速度档只写 NVS，不碰 OTA 分区；OTA 是固件维护流程。
实现位置components/debug_server、tools/ota_upload.py、partitions_ota_4mb.csv
验证方法BLE 发送 W1，电脑连接热点后访问 /api/ota/status。

S11小程序最小参数面板

小程序
策略小程序只做少量运动参数：Kp、Ki、Kd、speed.gear。复杂完整参数表保留给 Wi-Fi/JSON 后备诊断。
范围Kp 7000-11000、Ki 0-300、Kd 0-3000、档位 1-3。
实现位置wechat-miniprogram/pages/params/、components/debug_server/debug_server_ble.c
验证方法发送并保存后立即读回 G，显示值必须与发送值一致。

S12小程序循线控制流程

小程序安全
策略小程序循线面板使用三框流式状态图。一键自检 依次读取 telemetry、必要时清理故障、执行 arm_auto；只有进入 AUTO_ARMED 后才允许 开始循线。
运行开始循线 发送 start_auto，停止 发送 stop。
实现位置wechat-miniprogram/pages/params/params.js、params.wxml、params.wxss
验证方法真机调试中观察状态图从自检滑到可开始，再进入正在循线。

S13自动循线速度档位

速度
策略自动循线三档为最大协议线速度：一档 250 mm/s / Vx=0.25，二档 600 mm/s / Vx=0.6，三档 1000 mm/s / Vx=1.0。
Z 上限一档 8000 mdeg/cmd，二档 10000 mdeg/cmd，三档 10000 mdeg/cmd。
实现位置components/control/line_trace_controller.c
验证方法切换小程序档位并开始循线，telemetry 中 motion.linear_mm_s 不超过对应档位上限。

S14偏离中心自适应降速

速度
策略有效线越偏离中心 x4/x5，线速度越低。变化使用先慢后快的二次曲线，不是线性降速。
当前曲线中心附近保持档位最大速度；最外侧 offset 保留约 45% 档位速度。
实现位置adaptive_linear_speed_mm_s() in components/control/line_trace_policy.c
验证方法观察 telemetry：offset 小时 motion.linear_mm_s 接近档位上限；offset 大时逐步下降。

S15偏离中心自适应转向

转向
策略偏离中心越多，Z 角度增量越大。变化同样使用先慢后快的二次曲线。
当前曲线将 raw offset 归一化为 -1..1，再平方并保留方向后进入 PID。小偏离轻微修正，大偏离快速增大转向。
实现位置curved_error_from_offset() in components/control/line_trace_policy.c
验证方法观察 telemetry：小 offset 时 motion.angular_mdeg_s 不应立即撞限幅；大 offset 时应明显增大。

S16PID 参数范围

PID小程序
策略小程序只开放紧范围 PID，避免现场误输入导致车辆行为不可控。
范围Kp 7000-11000，Ki 0-300，Kd 0-3000。当前默认 Kp=9000、Ki=0、Kd=0。
实现位置components/param_store/param_store.c、wechat-miniprogram/pages/params/params.js
验证方法错误参数应弹出红色错误提示；合法参数保存后能读回。

S17丢线找回

找线安全
策略自动运行中全丢线不立即停车。线速度置零，Z 使用安全找线增量，并朝最后一次正常循线转向的反方向找回。
当前值linear_mm_s=0，search turn=6000 mdeg/cmd。
实现位置search_turn_mdeg()、set_search_output() in components/control/line_trace_policy.c
验证方法故意让线偏出传感器，telemetry 应显示 LINE_LOST、线速度为 0，并向反方向寻找。

S18传感器异常

传感器安全
策略自动运行中短暂传感器超时或解析异常不直接进入 FAULT，而是执行零线速度安全找线。非自动运行状态保持停车。
边界底盘发送失败仍进入 CHASSIS_SEND_FAILED。
实现位置components/control/line_trace_policy.c、components/control/line_trace_controller.c
验证方法运行中制造短暂传感器异常，车辆不应直接故障冲停；持续无法恢复时按后续安全策略扩展。

S19BLE 断连安全

安全无线
策略当前唯一需要立即停车的无线安全条件是 BLE 连接中断。若处于 AUTO_ARMED、AUTO_RUNNING 或 MANUAL_TEST，立刻发送底盘停止并回到 SAFE_IDLE。
实现位置components/debug_server/debug_server_ble.c
验证方法自动运行中断开 BLE，车辆应立即停车，telemetry 下次读取显示 SAFE_IDLE。

S20手动测试

调试安全
策略手动底盘测试是独立状态 MANUAL_TEST，只接受短时命令，有超时保护。
限制线速度上限 1500 mm/s，Z 上限 10000 mdeg/cmd，时长 50-3000 ms。
实现位置components/debug_protocol/debug_protocol.c、components/vehicle_state/vehicle_state.c
验证方法发送 manual_motion 后应自动回到调参状态或安全状态。

S21遥测观察

诊断
策略调试优先看 telemetry，不靠猜。关键字段包括 vehicle.motion_state、line.bits、line.offset、motion.linear_mm_s、motion.angular_mdeg_s、controller.line_phase、controller.pid_output_mdeg_s。
用途判断是 PID、限幅、速度、传感器、丢线找回方向还是底盘通信问题。
实现位置components/telemetry/telemetry.c
验证方法BLE JSON {"type":"get_telemetry"} 或 Wi-Fi /api/telemetry。

S22蓝牙设备选择与命名

小程序无线
策略小程序必须让用户看到并选择 BLE 设备，不静默自动连接。BLE 名称可完整修改，支持中文、英文、数字、-、_，限制 1-20 个 UTF-8 字节。
扫描默认优先筛选 DCar-Liner 设备，也可显示全部有名设备。手机侧最多保存 8 个已匹配名称作为扫描提示，不占用 ESP32 资源。
实现位置wechat-miniprogram/pages/params/、components/debug_server/debug_server_ble.c
验证方法修改名称后断开重扫，应能看到新名称并重新连接。
